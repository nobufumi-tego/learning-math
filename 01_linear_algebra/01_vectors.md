## 1. ベクトルとは何か — 「数の並び」

### 直感

ベクトルとは、**いくつかの数を一列に並べたもの**。それだけ。

```
v = (1, 2, 3)        ← 3つの数が並んだ「3次元ベクトル」
```

これに、いろんな解釈ができます:

| 解釈 | 例 |
|---|---|
| **位置** | 3次元空間の点 (x=1, y=2, z=3) |
| **移動** | 「右に1、上に2、奥に3 動く」 |
| **速度** | 「東に1, 北に2, 上に3 m/s で進んでいる」 |
| **特徴量** | あるユーザーの「年齢=1, 身長=2, 体重=3 のスケール値」 |
| **画像の1ピクセル** | (R=1, G=2, B=3) の色 |
| **単語の意味** | Word2Vec で「猫 = (0.1, -0.4, 0.9, ...)」のような数百次元 |

つまり、**ベクトル = この世界の何かを「数の並び」で表現する道具**です。

### 記号

数学では、ベクトルを太字 **`v`** や矢印 **`v⃗`** で書きます:

```
v = (v₁, v₂, ..., vₙ)        ← n 個の数からなる「n次元ベクトル」
```

n を**次元 (dimension)** と言います。
3次元なら3つの数、100次元なら100個の数。

### Python での表現

```python
# === 標準形式 (NumPy) ===
import numpy as np

v = np.array([1.0, 2.0, 3.0])  # shape: (3,) ← 3次元ベクトル
print(v.shape)                  # (3,)
print(v.ndim)                   # 1 (1次元配列)

# === JAX形式 ===
import jax.numpy as jnp

v_jax = jnp.array([1.0, 2.0, 3.0])  # shape: (3,)
```

> ⚠️ **shape の罠**: `(3,)` と `(3, 1)` と `(1, 3)` は別物。NumPy / JAX は **形状 (shape) のミスマッチ** が最頻出のバグ源。各セルで `print(x.shape)` する習慣を。

## 2. ベクトルの基本演算

### 加算 (足し算)

2つのベクトルを足すと、要素ごとに足される:

```
(1, 2, 3) + (4, 5, 6) = (5, 7, 9)
```

```python
import numpy as np

u = np.array([1, 2, 3])
v = np.array([4, 5, 6])
print(u + v)            # [5 7 9]
```

幾何学的には、「**矢印を継ぎ足して、終点まで歩く**」イメージ:

```
   ↗ (4,5,6)
  /
 / + 
↗ (1,2,3)
```

### スカラー倍

ベクトルを「3倍する」とは、全要素を3倍すること:

```
3 × (1, 2, 3) = (3, 6, 9)
```

```python
print(3 * u)            # [3 6 9]
print(0.5 * u)          # [0.5 1.  1.5]
print(-1 * u)           # [-1 -2 -3]   ← 反対向き
```

### 引き算

```
(4, 5, 6) − (1, 2, 3) = (3, 3, 3)
```

引き算は「**v から u に向かう矢印**」。
2点間の距離計算で頻出します。

```python
print(v - u)            # [3 3 3]
```

## 3. 内積 (dot product) — ベクトル界の主役

ベクトル同士の**かけ算には2種類**あります。先に重要な方、**内積**から。

### 定義

ベクトル `u = (u₁, ..., uₙ)` と `v = (v₁, ..., vₙ)` の内積:

```
u · v = u₁v₁ + u₂v₂ + ... + uₙvₙ
      = Σᵢ uᵢ vᵢ
```

つまり「**対応する成分をかけて、全部足す**」。
結果は **スカラー (1つの数)** になります（ベクトルじゃない）。

### 計算例

```
(1, 2, 3) · (4, 5, 6) = 1×4 + 2×5 + 3×6 = 4 + 10 + 18 = 32
```

### Python での内積

```python
# === 標準形式 (NumPy) ===
import numpy as np

u = np.array([1, 2, 3])
v = np.array([4, 5, 6])

# 4通りの書き方、全部同じ結果
print(u @ v)            # 32  ← @ 演算子（推奨）
print(np.dot(u, v))     # 32
print(np.inner(u, v))   # 32
print(np.sum(u * v))    # 32  ← 内積の定義そのもの

# === JAX形式 ===
import jax.numpy as jnp

u_j = jnp.array([1, 2, 3])
v_j = jnp.array([4, 5, 6])
print(u_j @ v_j)        # 32
print(jnp.dot(u_j, v_j))  # 32
```

### なぜ内積が重要か — 「**似ている度**」を測る道具

内積は、**2つのベクトルがどれくらい同じ向きを向いているか** を表します。

- **同じ向き** → 内積は大きな正の数
- **直角** → 内積は **0**
- **逆向き** → 内積は大きな負の数

これを使うと、

- ユーザーAとユーザーBの趣味の「似ている度」
- 文章Aと文章Bの内容の「似ている度」
- 画像Aと画像Bの特徴の「似ている度」

が**全部内積で計算できます**。

これがなぜ機械学習で内積がそこかしこに出てくるか、の理由です。

## 4. ノルム (norm) — ベクトルの「長さ」

### 定義

ベクトル `v = (v₁, ..., vₙ)` の **ユークリッドノルム**（普通の長さ）:

```
||v|| = √(v₁² + v₂² + ... + vₙ²)
      = √(v · v)
```

`|| ||` は「ノルム」を表す記号。**絶対値の双子バージョン** と思っておけばOK。

### これ、ピタゴラスの定理そのもの

2次元なら:
```
||(a, b)|| = √(a² + b²)
```
これ、`a² + b² = c²` から導かれる、**直角三角形の斜辺の長さ**。
3次元、4次元、…、n 次元に拡張したのがノルムです。

### 計算例

```
||(3, 4)|| = √(9 + 16) = √25 = 5
||(1, 2, 3)|| = √(1 + 4 + 9) = √14 ≈ 3.742
```

### Python

```python
import numpy as np

v = np.array([3.0, 4.0])
print(np.linalg.norm(v))             # 5.0
print(np.sqrt(v @ v))                # 5.0  ← 定義どおり

# 3次元
v3 = np.array([1.0, 2.0, 3.0])
print(np.linalg.norm(v3))            # 3.7416...

# === JAX形式 ===
import jax.numpy as jnp
print(jnp.linalg.norm(jnp.array([3.0, 4.0])))  # 5.0
```

### 単位ベクトル

ノルムが **1** のベクトルを **単位ベクトル (unit vector)** と言います。
任意のベクトルを「**自分のノルムで割る**」と単位ベクトルが作れます:

```
v̂ = v / ||v||
```

これを **正規化 (normalization)** と呼ぶ。
**「向きだけが意味のある量」** を扱いたいときに頻出。

```python
v = np.array([3.0, 4.0])
v_unit = v / np.linalg.norm(v)
print(v_unit)                        # [0.6 0.8]
print(np.linalg.norm(v_unit))        # 1.0
```

### いろんなノルム

実は「長さ」の測り方は**1つじゃない**:

| ノルム | 数式 | 意味 |
|---|---|---|
| **L²ノルム** (デフォルト) | `√Σ vᵢ²` | 普通の長さ、ユークリッド距離 |
| **L¹ノルム** (マンハッタン) | `Σ \|vᵢ\|` | 縦+横の道のり (碁盤目の街での距離) |
| **L∞ノルム** (最大) | `max\|vᵢ\|` | 最大成分の絶対値 |

機械学習では、**L¹** は「スパース化 (なるべく0を多くする) 」に、**L²** は「ふつうの正則化」によく使われます。

```python
v = np.array([3.0, -4.0, 5.0])
print(np.linalg.norm(v, ord=2))      # 7.07... (L²)
print(np.linalg.norm(v, ord=1))      # 12.0   (L¹)
print(np.linalg.norm(v, ord=np.inf)) # 5.0    (L∞)
```

## 5. 角度とコサイン類似度

### 内積 と 角度の関係

実は内積には、**幾何学的な公式**もあります:

```
u · v = ||u|| × ||v|| × cos θ
```

ここで `θ` は2つのベクトルのなす角度。

これを変形すると:

```
cos θ = (u · v) / (||u|| ||v||)
```

これを **コサイン類似度 (cosine similarity)** と呼びます。
- 値の範囲: **−1 〜 +1**
- 1 に近い → ほぼ同じ向き = 「似ている」
- 0 → 直角 = 「無関係」
- −1 → 逆向き = 「正反対」

### Python

```python
import numpy as np

def cosine_similarity(u: np.ndarray, v: np.ndarray) -> float:
    """2つのベクトルのコサイン類似度.

    Args:
        u, v: 同じ shape のベクトル

    Returns:
        −1 〜 +1 の値
    """
    return float(u @ v / (np.linalg.norm(u) * np.linalg.norm(v)))


# 例: 2人のユーザーの趣味ベクトル (映画5作品の評価, 5段階)
alice = np.array([5, 4, 1, 5, 2])
bob   = np.array([4, 5, 2, 5, 1])
carol = np.array([1, 1, 5, 1, 5])

print(cosine_similarity(alice, bob))    # 0.95...  ← 似てる
print(cosine_similarity(alice, carol))  # 0.45...  ← あまり似てない
```

これがまさに **Netflix の推薦システム** や **Spotify の楽曲推薦** の基本原理です。
あなたと「コサイン類似度の高い人」の好きなものを、あなたにもおすすめする。

### ChatGPT/Claude との接続

LLM が単語を理解するときに使う **Word Embedding** は、各単語を高次元ベクトル (例: 1536次元) に対応させます。
そして単語の意味の近さを「**コサイン類似度**」で測ります:

```
"king" と "queen" の cos類似度 → 0.85
"king" と "banana" の cos類似度 → 0.03
```

つまり、**LLMは内積で言葉の意味を測っている**のです。

## 6. 直交性 (orthogonality)

`u · v = 0` のとき、2つのベクトルは **直交している (orthogonal)** と言います。
要するに「**直角に交わっている**」。

```python
e1 = np.array([1, 0, 0])
e2 = np.array([0, 1, 0])
print(e1 @ e2)                       # 0  ← 直交
```

直交しているベクトルは「**まったく無関係な方向**」を向いている、と解釈できます。
- x軸とy軸: 直交
- 「身長」と「IQ」をベクトル化: ほぼ直交
- 「身長」と「体重」をベクトル化: 直交しない（相関あり）

機械学習で「**特徴を直交化する**」操作 (PCAなど) は、「**情報の重複をなくす**」ことに対応します。

## 7. 機械学習での使い方 — ベクトルが居る場所

### (1) データ = ベクトル

訓練データ1件 = 特徴量ベクトル:

```python
# あるサンプル: [年齢, 年収, 学歴年数, 職歴年数]
sample = np.array([35, 600, 16, 12])  # shape: (4,)
```

データセット全体 = 行列 (= ベクトルの並び):
- 行: 各サンプル
- 列: 各特徴量
（行列は次のページ）

### (2) パラメータ = ベクトル

ニューラルネットワークの重み `w`、バイアス `b` は、ベクトルとして保持されます:

```python
w = np.array([0.5, -0.3, 0.7, 0.1])  # 学習で変化していく
```

### (3) 予測 = 内積

線形モデルの予測式:

```
ŷ = w · x + b
```

`w` と `x` の内積を取り、バイアスを足す。これだけ。
**ニューラルネットの1ニューロンの計算は、結局これ**。

```python
def linear_predict(w: np.ndarray, x: np.ndarray, b: float) -> float:
    """線形モデルの予測値."""
    return float(w @ x + b)


w = np.array([0.5, -0.3, 0.7])
x = np.array([1.0, 2.0, 3.0])
b = 0.1
print(linear_predict(w, x, b))   # 2.1
```

### (4) 損失 = ベクトルの距離

「予測が正解からどれくらい離れているか」を計算するときも、ベクトルの距離 (ノルム) を使います:

```python
y_true = np.array([1.0, 2.0, 3.0])
y_pred = np.array([1.1, 1.9, 3.2])

# 予測誤差 (差ベクトル)
error = y_pred - y_true
print(error)                          # [0.1 -0.1 0.2]

# L²ノルム (≒MSEのルート)
print(np.linalg.norm(error))          # 0.244...
```

## まとめ

| 概念 | 数式 | Python (NumPy / JAX) | 意味 |
|---|---|---|---|
| ベクトル | `v = (v₁, ..., vₙ)` | `np.array([...])` / `jnp.array([...])` | 数の並び |
| 加算 | `u + v` | `u + v` | 矢印を継ぎ足す |
| スカラー倍 | `αv` | `alpha * v` | 長さを変える |
| 内積 | `u · v = Σ uᵢvᵢ` | `u @ v` | 似ている度 |
| ノルム (L²) | `\|\|v\|\| = √(v·v)` | `np.linalg.norm(v)` | 長さ |
| 単位ベクトル | `v / \|\|v\|\|` | `v / np.linalg.norm(v)` | 向きだけ |
| コサイン類似度 | `(u·v)/(\|\|u\|\| \|\|v\|\|)` | (関数を作る) | 角度のcos |
| 直交 | `u · v = 0` | `u @ v == 0` | 90度 |

**この章のキー**: ベクトルとは数の並びで、内積でその「似ている度」を測れる。
これさえ握れば、機械学習の半分は分かったようなものです。
