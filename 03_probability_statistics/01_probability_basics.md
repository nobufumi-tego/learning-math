## 1. 確率とは — 「起こりやすさ」を 0〜1 で測る

### 直感

確率 `P(A)` は **「事象 A の起こりやすさ」を 0〜1 の数値で表したもの**:

- `P(A) = 0` → 絶対に起こらない
- `P(A) = 0.5` → 半々の確率
- `P(A) = 1` → 必ず起こる

### 例

| 事象 | 確率 |
|---|---|
| サイコロで 1 が出る | 1/6 ≈ 0.167 |
| コインで表が出る | 1/2 = 0.5 |
| トランプで♠の Ace を引く | 1/52 ≈ 0.019 |
| 明日地震が起こる | (難しい問題) |

最後の例のように、**「同じ条件で繰り返せない事象」** の確率は、頻度ではなく**ベイズ的な「信念の強さ」**として扱います (詳しくは [`04_bayes.md`](04_bayes.md))。

---

## 2. 確率の 3 つの公理 (Kolmogorov)

数学的に確率は、以下の 3 つの公理を満たすものとして定義されます:

1. `P(A) ≥ 0` (確率は非負)
2. `P(全体) = 1` (全事象の確率は 1)
3. `P(A ∪ B) = P(A) + P(B)` (互いに排反な事象なら和)

これだけのシンプルなルールから、確率論のすべてが導かれます。

## 3. Python で「確率」を観察 — 大数の法則

理論上 `P(表) = 0.5` のコインを、**実際に投げてみる** とどうなる?

```python
import numpy as np

rng = np.random.default_rng(seed=42)

for n in [10, 100, 1_000, 100_000, 10_000_000]:
    flips = rng.integers(0, 2, size=n)   # 0 or 1
    ratio = flips.mean()                  # 1 (=表) の割合
    print(f'n = {n:>10}  →  表の割合: {ratio:.6f}')
```

出力例:
```
n =         10  →  表の割合: 0.700000
n =        100  →  表の割合: 0.480000
n =       1000  →  表の割合: 0.508000
n =     100000  →  表の割合: 0.500300
n =   10000000  →  表の割合: 0.499838
```

**試行回数 n を増やすほど、理論値 0.5 に近づく** ことが分かります。
これを **大数の法則** と呼びます。確率論の最も基礎的な定理。

## 4. 同時確率・条件付き確率・独立性

### 同時確率 `P(A ∩ B)`

「**A と B が両方起こる確率**」。

例: トランプで 1 枚引いて「♠ かつ 数字が偶数」の確率 = 6/52 (♠ の 2,4,6,8,10,Q ※絵札を数字に含むなら別計算)

### 条件付き確率 `P(A | B)`

「**B が起きたという情報のもとで、A が起こる確率**」。

```
P(A | B) = P(A ∩ B) / P(B)
```

読み方: "P of A given B"

### 例: 上司が休みの日に風邪をひく確率

ある会社で:
- `P(上司休み) = 0.05`
- `P(上司風邪) = 0.03`
- `P(上司休み ∩ 風邪) = 0.025`

**「休んだ日に風邪をひいてた確率」**:
```
P(風邪 | 休み) = P(休み ∩ 風邪) / P(休み)
              = 0.025 / 0.05
              = 0.5
```

「**休んだ日の半分は、実は風邪が原因だった**」と分かる。

### 独立性

事象 A と B が **独立** とは:

```
P(A ∩ B) = P(A) × P(B)
```

または同じことだけど、

```
P(A | B) = P(A)        ← B が起きても A の確率は変わらない
```

例: コインを 2 回投げて両方表 → `P(表 ∩ 表) = 1/2 × 1/2 = 1/4` (1 回目は 2 回目に影響しない)

---

## 5. Python で確認

### 同時確率と独立性

```python
import numpy as np

rng = np.random.default_rng(seed=42)
n = 1_000_000

# 2 つのコインを独立に投げる
coin1 = rng.integers(0, 2, size=n)
coin2 = rng.integers(0, 2, size=n)

# P(両方表)
both_heads = ((coin1 == 1) & (coin2 == 1)).mean()
print(f'P(両方表) ≈ {both_heads:.4f}  (理論 0.25)')

# 確認: P(coin1=表) × P(coin2=表)
print(f'P(coin1) × P(coin2) = {coin1.mean() * coin2.mean():.4f}')
```

### 条件付き確率

```python
# 2 つのサイコロ
d1 = rng.integers(1, 7, size=n)
d2 = rng.integers(1, 7, size=n)
total = d1 + d2

# P(合計 = 7 | d1 = 3) はどう?
mask_d1_is_3 = (d1 == 3)
mask_total_7 = (total == 7)

p_given = mask_total_7[mask_d1_is_3].mean()
print(f'P(合計=7 | d1=3) ≈ {p_given:.4f}  (理論 1/6 = {1/6:.4f})')
# d1=3 のとき、合計が 7 になるには d2=4 のみ → 1/6
```

## 6. 排反 vs 独立 — よくある混同

| 用語 | 意味 |
|---|---|
| **排反** (mutually exclusive) | 同時には起きない (`P(A ∩ B) = 0`) |
| **独立** (independent) | お互い影響しない (`P(A ∩ B) = P(A) P(B)`) |

これらは **全然別の概念**:

- サイコロで「1 が出る」と「2 が出る」 → **排反** (同時には起きない)
- 2 個のサイコロで「1 個目が 1」と「2 個目が 2」 → **独立** (お互い影響しない)

排反な 2 事象は **独立じゃない** (1 つが起きると相手は起きないので、影響している)。

---

## 7. 包除原理 (inclusion-exclusion)

`A ∪ B` (どちらか少なくとも 1 つ起きる) の確率:

```
P(A ∪ B) = P(A) + P(B) − P(A ∩ B)
```

「2 つ足したら、両方起こる場合を 2 重カウントしてるので引く」。
ベン図を描けば一目瞭然。

3 つ以上に一般化したのが **包除原理**。

## 8. ML / AI への接続

### 確率は ML の出力そのもの

分類モデル (画像認識など) は最終的に確率を出します:

```python
# 例: 画像分類モデルの出力
probs = {
    "cat": 0.85,
    "dog": 0.10,
    "bird": 0.04,
    "other": 0.01,
}
# → 合計 1.0、これが確率分布
```

ソフトマックスで正規化されたものが「クラス確率」。

### 独立性の仮定でモデルを単純化

ナイーブベイズ分類器は「**特徴量が条件付きで独立**」 という (大胆な) 仮定で動きます:
```
P(class | x₁, x₂, ..., xₙ) ∝ P(class) · ∏ P(xᵢ | class)
```

仮定は強いけど、スパムフィルタ等で**驚くほど効く**。

---

## 9. ハマりポイント

- **独立 ≠ 排反**: 違うものなので注意
- **小サンプルで確率を語らない**: 大数の法則が効くまで n が必要 (1万以上が安全)
- **条件付き確率の方向**: `P(A|B)` と `P(B|A)` は別物 (これがベイズの定理の出発点)
- **「奇跡」と「確率」**: 確率 1/100万 でも、100万人中 1 人は経験する

## まとめ

| 概念 | 数式 | Python (シミュレーション) |
|---|---|---|
| 確率 | `P(A) ∈ [0, 1]` | `(events).mean()` |
| 同時確率 | `P(A ∩ B)` | `((A) & (B)).mean()` |
| 条件付き確率 | `P(A\|B) = P(A∩B) / P(B)` | `A[B].mean()` |
| 独立 | `P(A∩B) = P(A) P(B)` | – |
| 大数の法則 | n→∞ で頻度 → 確率 | n を増やして検証 |

**この章のキー**: 確率 = 0〜1 の数値。条件付き確率と独立性は別物。Python でシミュレーションすれば理論が腹落ちする。
