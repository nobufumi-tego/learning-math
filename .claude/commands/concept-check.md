# 概念理解度チェックコマンド

対象概念: `$ARGUMENTS`

ユーザーがこの概念を本当に理解しているかを、**ソクラテス式問答**で確認してください。

## 進行ルール（厳守）

1. **答えを先に教えない**
   - まず質問を投げて、ユーザーに考えさせる
   - ユーザーが答えてから、正誤を判定する

2. **段階的に難易度を上げる**
   - レベル1: 定義の確認（言葉で答えられるか）
   - レベル2: 記号での表現（数式で書けるか）
   - レベル3: 具体例の計算（数字を入れて計算できるか）
   - レベル4: Python 実装（コードを書けるか）
   - レベル5: 応用（ML/AI のどの場面に出てくるか）

3. **ヒントは段階的に**
   - 最初は質問だけ
   - 詰まったら小さなヒント
   - それでも詰まったら答えと解説

4. **必ず1問ずつ**
   - 一度に複数の質問を出さない
   - 1問終わったら次に進む前に正誤フィードバック

5. **最後に診断**
   - 5問すべて終わったら、レベル別に理解度を診断
   - 弱い箇所を明示して、次に学ぶべきことを提案

## 出力例

```
=== 概念チェック: <対象> ===

【Q1 / レベル1: 定義】
<対象> を一言で説明してください。

(ユーザーの回答を待つ)
```

回答後:

```
【判定】 ✅正解 / ⚠️惜しい / ❌要確認

【補足】
<追加説明>

【Q2 / レベル2: 記号】
<次の質問>
```
