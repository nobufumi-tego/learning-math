## 1. Jupyter Lab とは何か

ひと言で言うと:

> **「ブラウザの中で動く、メモ帳 + 電卓 + Python の作業環境」**

- 見た目は Web サイトですが、**自分の PC の中で動いています** (インターネットには出ていません)
- 文章 (説明・数式) と Python コードを **同じページに並べて書ける**
- コードを実行すると、**結果がその場に表示** される (グラフも表も)

これを 1 つにまとめたファイルを **notebook (ノートブック)** と呼びます。拡張子は `.ipynb`。

ブラウザのタブを閉じても、**ファイル自体はディスクに保存** されているので消えません。

## 2. 画面の見方

起動直後の Jupyter Lab はだいたい以下のレイアウトです:

```
┌────────────────────────────────────────────────────────────┐
│ File  Edit  View  Run  Kernel  Tabs  Settings  Help        │  ← メニューバー
├────────┬───────────────────────────────────────────────────┤
│        │                                                   │
│ 📁 src │   [開いている notebook / md ファイル]              │
│ 📁 ... │                                                   │
│        │   ┌──────────────────────────────┐               │
│ 📄 RE  │   │ # 見出し                     │  ← セル        │
│ 📄 ... │   └──────────────────────────────┘               │
│        │                                                   │
│ ファイ  │   ┌──────────────────────────────┐               │
│ ル一覧  │   │ x = 1 + 2                    │  ← Code セル  │
│        │   │ print(x)                     │               │
│        │   └──────────────────────────────┘               │
│        │   3                                ← 実行結果     │
│        │                                                   │
└────────┴───────────────────────────────────────────────────┘
```

| エリア | 役割 |
|---|---|
| **左サイドバー** | ファイル一覧 (フォルダを辿って .ipynb / .md を開く) |
| **中央** | 開いている notebook や md ファイルが**タブ**で並ぶ |
| **メニューバー** | File / Edit / Run / Kernel / Settings 等 |
| **右下** | カーネル状態 (Python が動いているか / 待機中か) |

> 💡 **タブ** は複数の notebook を同時に開けます。ブラウザのタブと似た感覚。

## 3. セルとは — 実行の最小単位

notebook (`.ipynb`) を開くと、**セル (cell)** という横長の枠がいくつも縦に並んでいます。

**セル = 1 つの実行単位**。これが Jupyter の核心です。

```
┌────────────────────────────────┐
│ # 円の面積を計算する             │   ← Markdown セル (説明文)
└────────────────────────────────┘

┌────────────────────────────────┐
│ import math                     │   ← Code セル (Python コード)
│ r = 5                           │
│ area = math.pi * r ** 2         │
│ print(area)                     │
└────────────────────────────────┘
78.53981633974483                    ← 実行すると結果がここに出る
```

- セルをクリックすると、そのセルが **選択状態** になる (枠の左に色が付く)
- Code セルは Python が動く / Markdown セルは文章として表示される
- 上から順に実行するのが基本 (変数は引き継がれる)

## 4. セルを実行する — Shift+Enter

これが **最重要のショートカット** です。

### 手順 (3 ステップ)

1. 実行したいセルを **マウスでクリック** (枠が青くなる = 選択)
2. キーボードで **Shift + Enter** を同時押し
3. 結果がセル直下に表示され、**自動的にカーソルが次のセルへ移動** する

→ あとはひたすら **Shift+Enter** を押し続ければ、上から順番に notebook 全体が実行できます。

### 似たショートカット

| キー | 効果 | 使いどころ |
|---|---|---|
| **Shift+Enter** ⭐ | 実行 + 次のセルへ移動 | 一番よく使う、上から順に進める時 |
| **Ctrl+Enter** | 実行 + そのセルに留まる | 同じセルを何度も試したい時 (パラメータ調整等) |
| **Alt+Enter** | 実行 + 下に新しい空セル追加 | コードをどんどん書き足したい時 |

### 実行中の表示

- セルの左の `[ ]` が `[*]` に変わる → **実行中**
- 実行が終わると `[1]`, `[2]`... の番号が入る (= 何番目に実行されたか)
- これで「**今このセルが動いてる**」 と分かります

> 💡 重い計算が走ると `[*]` のまま数秒〜数分待つことがあります。フリーズではないので待ちましょう。

## 5. Code セルと Markdown セル

セルには 2 種類あります。

| 種類 | 役割 | 中身の例 |
|---|---|---|
| **Code セル** | Python を実行する | `print("hello")` → 結果が下に出る |
| **Markdown セル** | 文章・数式・表を書く | `# 見出し` → 見出しに整形される |

### 種類の切り替え

セルを選択した状態で:
- **Y** キー: そのセルを **Code セル** に変換
- **M** キー: そのセルを **Markdown セル** に変換

> ⚠️ ただし、これは **コマンドモード** (セルを選択しただけで中身を編集していない状態) のときだけ効きます。
> セル内の文字列にカーソルが入っている (編集モード) ときに M を押すと、ただのアルファベット入力になります。
> **Esc** キーで編集モード → コマンドモードに切替できます。

### 数式の書き方 (Markdown セル)

```
$f(x) = x^2$           → インライン: f(x) = x²
$$\int_0^1 x\,dx$$     → 中央寄せ大きい数式
```

LaTeX 記法が使えます。本リポジトリの md / notebook はこの形式で数式を書いています。

## 6. ファイルの種類

Jupyter Lab で開ける主なファイル:

| 拡張子 | 種類 | 開いた時の動作 |
|---|---|---|
| `.ipynb` | **Jupyter Notebook** | セルが並んだ notebook ビュー |
| `.md` | **Markdown** | このリポジトリでは **プレビュー** で開く設定 (整形表示) |
| `.py` | **Python ファイル** | テキストエディタとして開く |
| `.txt`, `.json`, `.yaml` 等 | テキスト系 | テキストエディタとして開く |
| `.png`, `.jpg` 等 | 画像 | ビューアで開く |

### このリポジトリの典型的な使い方

1. 左ファイルツリーで **章フォルダ** (例: `01_linear_algebra/`) を開く
2. `README.md` をクリック → **解説をプレビューで読む**
3. 解説中のリンクから `notebooks/01_vectors.ipynb` を開く
4. 上のセルから **Shift+Enter** で順番に実行
5. コードを編集してみる → 再度 **Shift+Enter** で結果を確認

→ **読む (md) と動かす (ipynb) を行き来** するのがこのリポジトリの学習スタイルです。

## 7. 困ったときの対処法

### 🔴 セルが止まらない (実行中マーク `[*]` が永遠に消えない)

- メニュー: **Kernel → Interrupt Kernel** (実行を中断)
- それでもダメなら: **Kernel → Restart Kernel** (Python を再起動、変数は全部消える)

### 🔴 「変なエラーが続けて出る」「何かおかしい」

`Kernel → Restart Kernel and Run All Cells` で **全セルを最初から実行し直す**。
これでだいたい治ります。

### 🔴 ブラウザのタブを間違えて閉じてしまった

ファイル自体は保存されています。**ターミナル / ウィンドウは閉じない** で、ブラウザで http://localhost:8888 を開けば戻れます。
完全に閉じてしまったら、もう一度 `start.bat` / `./start.sh` を実行。

### 🔴 「Port 8888 is already in use」 エラー

前回の Jupyter Lab が **完全に停止していない** 状態です。
- Mac/Linux: `pkill -f jupyter`
- Windows: タスクマネージャ で `python.exe` を終了
- 再度 start スクリプトで起動

### 🔴 コードを書き換えたのに結果が変わらない

セルを **再実行** していますか? 編集しただけでは結果は変わりません。
**Shift+Enter** で再度実行してください。

## 8. 保存とバックアップ

### 保存

- メニュー: **File → Save Notebook** (または **Ctrl+S** / Mac は **⌘+S**)
- Jupyter Lab には **自動保存** もあります (約 2 分ごと)
- ファイル名横に **● (黒丸)** が出ていたら未保存、消えていたら保存済み

### Checkpoint

「Save Notebook」 すると **チェックポイント** が作られ、メニュー **File → Revert Notebook to Checkpoint** で戻れます。

### git でバックアップ (上級者向け)

定期的に `git add . && git commit -m "学習記録"` するとさらに安心です。

## 9. 起動コマンドの使い分け

通常は **start スクリプト** ([`start.bat`](../start.bat) / [`start.sh`](../start.sh) / [`start.ps1`](../start.ps1)) を使えば OK ですが、ピンポイントで違うコマンドを使いたい場面もあります。

### 使い分け早見表

| 場面 | コマンド |
|---|---|
| **普段の起動** (大半のケース) | `start.bat` (Win) / `./start.sh` (Mac/Linux) |
| **特定の notebook を直接開きたい** | `uv run lab.py 02_calculus/notebooks/02_derivatives.ipynb` |
| **README を開かず素の Jupyter Lab を出したい** | `uv run lab.py` |
| **最速で起動** (依存チェックすら省略) | `uv run jupyter lab` |
| **特殊オプション付き** (ポート変更等) | `uv run jupyter lab --port=8889 --no-browser` |

### 性能差はほぼ誤差

| コマンド | 2 回目以降の起動時間 |
|---|---|
| `uv run jupyter lab` | 約 5〜10 秒 |
| `uv run lab.py` | 約 5〜10 秒 |
| `./start.sh` (start scripts 経由) | 約 7〜13 秒 |

→ **誤差レベルなので「いつも start」 で OK**。
ピンポイントで開きたいときだけ `uv run lab.py <ファイル>` を覚えておけば十分です。

## 10. ショートカット早見表

これだけ覚えておけば 95% 困りません:

| キー | 効果 |
|---|---|
| **Shift + Enter** ⭐ | セル実行 + 次のセルへ |
| **Ctrl + Enter** | セル実行 + その場に留まる |
| **Alt + Enter** | セル実行 + 下に新セル |
| **Esc** | 編集モード → コマンドモード |
| **Enter** | コマンドモード → 編集モード (中身を書く) |
| **A** (コマンドモード) | 上に新しいセルを追加 |
| **B** (コマンドモード) | 下に新しいセルを追加 |
| **D, D** (D を 2 回) | 選択中のセルを削除 |
| **Z** (コマンドモード) | セル削除を取り消し (Undo) |
| **M** (コマンドモード) | セルを Markdown に変換 |
| **Y** (コマンドモード) | セルを Code に変換 |
| **Ctrl + S** (Mac は **⌘+S**) | 保存 |
| **Ctrl + Shift + P** | コマンドパレット (全機能を検索) |

> 💡 **コマンドモード** = セルが青枠 (中身を触っていない)
> **編集モード** = セルが緑枠 (中身を書いている)
> **Esc / Enter** で行き来できます
