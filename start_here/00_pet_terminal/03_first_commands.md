# 03. 最初の3つのおまじない — pwd・ls・cd

```
       ／￣￣＼
      |  ・ω・  |   さて、今日はぼくにお散歩を教えてもらうよ。
      |   ⌒    |   3つの命令だけ、覚えてくれる？
       ＼____／
```

このページで覚えるのは、たった3つ:

| 命令 | 意味 | 例えるなら |
|---|---|---|
| `pwd` | 今どこにいる？ | 「ペンタ、今どの部屋？」 |
| `ls` | 周りに何がある？ | 「周りに何があるか教えて」 |
| `cd` | 移動する | 「あの部屋に行って」 |

この3つだけで、コンピュータの中を**完全に歩き回れる**ようになります。

## まずはコンピュータの中を「家の構造」だと思ってみる

コンピュータの中のフォルダ（ディレクトリ）は、**家の中の部屋**みたいなもの:

```
/                       ← 家全体 (ルート、玄関)
├── home/               ← 住人たちの部屋
│   └── nobufumi/       ← あなたの部屋
│       ├── Documents/  ← 書類棚
│       ├── Pictures/   ← アルバム棚
│       ├── Music/      ← 音楽コレクション
│       └── leaning-math/  ← このリポジトリ
└── etc/                ← 設定資料室
```

ペンタは家の中のどこかに必ず立っています。
「今、書類棚の前にいる」のように、**居場所**を持っています。

## 命令 1: `pwd` — 今どこ？

```bash
pwd
```

返事 (例):
```
/home/nobufumi
```

これは「**ペンタ、今あなたの部屋にいるよ**」という意味。

`pwd` は **Print Working Directory** の略。「**今いる作業ディレクトリを表示せよ**」という命令です。
迷子になったら、まず `pwd`。

### ホームディレクトリと「~」

`/home/nobufumi` のような **「あなたの部屋」** を **ホームディレクトリ** と呼びます。
ターミナルでは、これを **`~` (チルダ)** という記号で短く書けます。

```
~ = /home/nobufumi      (Linux)
~ = /Users/nobufumi      (Mac)
~ = C:\Users\nobufumi    (Windows)
```

`~` は「**家**」のイメージそのまま。「ペンタ、家に帰って」と言いたいときは:

```bash
cd ~
```

または引数なしで:

```bash
cd
```

これで一瞬で家に戻れます。

## 命令 2: `ls` — 周りに何がある?

```bash
ls
```

返事 (例):
```
Documents  Pictures  Music  leaning-math
```

「ペンタの周りに、これだけのフォルダや書類があるよ」という意味。

`ls` は **list** の略。**「並べて見せろ」** です。

### `ls` のオプション

`ls` にちょっとオマケをつけると、もっと教えてくれます:

```bash
ls -l    # 詳しく (サイズ、更新日時も)
ls -a    # 隠しファイルも
ls -lh   # サイズを人間に読みやすい単位で
```

`-l` `-a` `-h` のような、`-` で始まるものを **オプション** と呼びます。
ペンタへの「もっとこうしてね」というお願いです。

例 (`ls -lh` の結果):
```
total 16K
drwxr-xr-x  3 nobufumi staff  102 Jan  3 10:00 Documents
drwxr-xr-x  5 nobufumi staff  170 Jan  3 12:30 Pictures
-rw-r--r--  1 nobufumi staff  2.1K Jan  3 09:15 memo.txt
```

最初の文字 `d` か `-` で、フォルダかファイルかが分かります:
- `d` で始まる → ディレクトリ (フォルダ)
- `-` で始まる → 普通のファイル

詳しい意味は今は気にしなくてOK。「**そういうものだ**」と思っておきましょう。

## 命令 3: `cd` — 移動する

```bash
cd Documents
```

これで「**ペンタ、Documents の部屋に入って**」という命令。
何も返事は無いけど、ちゃんと中に入っています。確認してみよう:

```bash
pwd
```
返事:
```
/home/nobufumi/Documents
```

ほら、移動した。

`cd` は **Change Directory** の略。**「ディレクトリを変える」** です。

### よく使う `cd` のパターン

```bash
cd Documents       # 中の Documents へ
cd ..              # 一つ上 (親フォルダ) へ
cd ../..           # 二つ上へ
cd /               # ルート (家全体) へ
cd ~               # ホームに戻る
cd                 # 同じく、ホームへ
cd -               # さっきいた場所に戻る (ペンタの記憶力)
```

### 「`.`」と「`..`」の意味

| 記号 | 意味 |
|---|---|
| `.` | **今いる場所** |
| `..` | **一つ上の場所 (親)** |

`.` は「ここ」、`..` は「親」と覚えておけば OK。
よく使うので、ペンタに餌をあげるくらい大事な記号。

## やってみる: 散歩のシナリオ

ターミナルを開いて、こう打ってみましょう (Mac/Linux 想定):

```bash
pwd                    # 1. 今どこ？
ls                     # 2. 何がある？
cd ~                   # 3. ホームに戻る
ls                     # 4. ホームに何がある？
cd Documents           # 5. 書類部屋へ (なければ別のフォルダで)
pwd                    # 6. 移動できた？
cd ..                  # 7. 親に戻る
pwd                    # 8. ホームに戻ってる？
```

8回の命令で、ペンタを家の中で散歩させたことになります。
これだけで、もうあなたは「**ターミナルを使える人**」の仲間入り。

## オートコンプリート — Tabキーが救世主

長いフォルダ名を全部打つのは面倒ですよね。
そんなとき、**Tab キー** を押すと、ペンタが残りを補完してくれます。

```bash
cd Doc[Tab]      # → cd Documents/ になる
```

「Doc」まで打って Tab を押すと、ペンタが「Documents だね？」と察してくれる。
**プロは Tab を毎秒押している**と言っても過言じゃない。

## 矢印キー (↑↓) — 過去の命令を呼び出す

打った命令は、ペンタが履歴に覚えています。
**↑キー** で過去の命令、**↓キー** でその逆。

何度も同じことを打つ前に、まず ↑ を押してみる。
これも**プロの基本動作**です。

## まとめ — ターミナルの「歩き方」3つ

| 命令 | 意味 | 略 |
|---|---|---|
| `pwd` | 今どこ？ | Print Working Directory |
| `ls` | 周りに何が？ | LiSt |
| `cd <場所>` | 移動 | Change Directory |

便利キー:
- **Tab** = 補完
- **↑↓** = 過去の命令
- **Ctrl+C** = 命令を中断 (緊急ストップ)

## 🔍 ググってみよう

- **POSIX** — UNIX 系 OS の標準。pwd・ls・cd は POSIX で定義された命令
- **絶対パス vs 相対パス** — `/home/nobufumi/Documents` と `Documents` の違い
- **シンボリックリンク** — フォルダのショートカット
- **ホームディレクトリ** — `~` の正体
- **ファイルシステム** — Linux の ext4、macOS の APFS、Windows の NTFS
- **Filesystem Hierarchy Standard (FHS)** — Linux のフォルダ構造の取り決め
- **ターミナルショートカット** — Ctrl+A, Ctrl+E, Ctrl+R などで効率爆上げ
