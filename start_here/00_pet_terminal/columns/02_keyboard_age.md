# コラム 02: キーボードがマウスより速い理由 — タイピングの王様

> *マウスを使う人が遅く見える瞬間、わかりますよね。*

## マウス vs キーボード — 数字で見る

「キーボードの方が速い」って、なんとなく言われますよね。
実際、どれくらい違うのか?

ある実験 (Bruce Tognazzini, Apple のUI研究者の調査) によると:
- **キーボードショートカット**: 平均 1.5秒
- **マウス操作**: 平均 2.5秒

マウスは1動作で **1秒遅い**。
1日に100回操作すると、**100秒の差** = ほぼ2分。
1年で **12時間** の差。

そして体感的にはもっと大きい差を感じます。
理由は次の3つ:

### 理由 1: 手をホームポジションから離さない
キーボードからマウスに手を伸ばす動作は、思っているより**集中力を奪います**。
プログラマー・ライター・編集者は、できるだけ手を移動させたくない。

### 理由 2: 思考と入力の遅延が小さい
マウスは「**マウスを動かして、目で位置を確認して、クリック**」と、3ステップ。
キーボードは「**指で押す**」の1ステップ。

### 理由 3: 並列処理ができる
タイピングは10本の指を同時に使える。
マウスは右手の1点のみ。

## 「マウスは遅い」を初めて気付いた人々

1980年代、プログラマー文化のなかで「**マウスより速い操作系**」が次々と生まれました。

### Vim (1991)
ビム。テキストエディタ。
- カーソル移動を `h j k l` キーで
- マウスを一切使わない設計
- 「**指先がホームポジションから動かない**」のが正義

学習コストは高いけど、習得すると **編集速度が爆発** します。
今でも世界中のプロが愛用。

### Emacs (1976)
エマックス。これも超古参のエディタ。
- 「**Ctrl + 何か**」「**Alt + 何か**」のショートカット祭り
- 「Emacs 教団」と呼ばれるほど信者が多い

Vim と Emacs の **エディタ戦争** は、プログラマー文化の伝説。
SNSで「Vim派」「Emacs派」をネタに語り合うのが、長年の風物詩。

### tmux / screen
ターミナルマルチプレクサ。
- 1つのターミナル画面を**分割**したり、**切り替え**したり
- 全部キーボードで完結

リモートサーバーで作業するエンジニアの**必須ツール**。

## キーボードショートカットの基本セット

OS問わず、覚えておくと**人生が楽**になるショートカット:

| 操作 | Mac | Windows/Linux |
|---|---|---|
| コピー | `⌘C` | `Ctrl+C` |
| ペースト | `⌘V` | `Ctrl+V` |
| 元に戻す | `⌘Z` | `Ctrl+Z` |
| 全選択 | `⌘A` | `Ctrl+A` |
| 検索 | `⌘F` | `Ctrl+F` |
| 別タブ | `⌘T` | `Ctrl+T` |
| タブ切替 | `⌘⇥` | `Ctrl+Tab` |
| アプリ切替 | `⌘⇥` | `Alt+Tab` |
| ターミナル | `⌘Space → terminal` | `Win → cmd` |

これだけで、マウス使用率を**激減**できます。

## ターミナルでのキー操作

ターミナルにも、強力なキーボード文化があります:

| キー | 効果 |
|---|---|
| `Tab` | 補完 |
| `↑` `↓` | 過去のコマンド |
| `Ctrl+A` | 行頭へ |
| `Ctrl+E` | 行末へ |
| `Ctrl+K` | カーソルから行末まで削除 |
| `Ctrl+U` | カーソルから行頭まで削除 |
| `Ctrl+W` | 1単語削除 |
| `Ctrl+R` | 過去のコマンドを検索 |
| `Ctrl+L` | 画面クリア (`clear` と同じ) |
| `Ctrl+C` | 実行中の処理を中断 |
| `Ctrl+D` | EOF (ログアウト・抜ける) |

**Ctrl+R** は神。ぜひ試してみてください。
過去に打ったコマンドを、文字列で検索できます。

## 「マウスを使わないことのオシャレさ」

エンジニア文化では、「**マウスを使わずに作業を完結する**」のが**カッコイイ**とされます。

理由は:
- 速い (実利)
- 集中力が切れない (実利)
- でかい画面のサーバーモニタの前で**カチャカチャやってる姿が様**になる (見栄)
- 周りに「あの人プロっぽい」と思わせる (見栄)

実利と見栄、半々です。
でも、マウスを使わないでサクサクと作業する姿は、本当に**忍者っぽくてカッコイイ**。

## キーボードの歴史

### QWERTY 配列 (1873年)
今のキーボードのキー配列「**QWERTY**」は、なんと **1873年のタイプライター由来**。
当時、タイピストが速く打ちすぎて**機械が絡まる**ので、わざと打ちにくくしたという伝説があります (諸説あり)。

つまり、あなたが今打っているキー配列は、**150年前の機械の制約**を引きずっています。
人類はそれを変えられないまま、ここまで来ました。

### Dvorak 配列 (1936年)
「QWERTY は遅すぎる」と、もっと速い配列を作った人がいます。
**ドヴォラック配列**。
理論上は QWERTY より速いとされますが、**慣性に勝てず**普及しませんでした。

### 日本語入力 (かな入力 vs ローマ字入力)
日本では「**かな入力**」と「**ローマ字入力**」の二派に分かれます。
かな入力のほうが理論上は速いけど、**ローマ字入力派が圧倒的多数**。

これも「**標準化の力**」というやつです。
速さより、**みんなが使ってる方が勝つ**。

## キーボードオタクの世界

エンジニアの中には、キーボードに**異常に投資する**人々がいます:

- **HHKB** (Happy Hacking Keyboard) — 日本製、3万円超、ファン多数
- **REALFORCE** — 静電容量無接点方式、打鍵感最高
- **Kinesis Advantage** — 凹型のエルゴノミクス、見た目がヤバい
- **自作キーボード** — 部品から組み立てる人々
- **メカニカルキーボード** — 軸 (Cherry MX 青軸/茶軸/赤軸...) で論争

「**3万円のキーボード**」と聞くと驚くかもしれませんが、
プロにとってキーボードは **毎日 8 時間使う商売道具**。
3万円を5年使えば、1日あたり16円。**コスパ最強**です。

## AI 時代でもキーボードは王様か?

「AIに音声で命令する時代になれば、キーボードはいらなくない?」と思うかもしれません。

でも、現状:
- **正確な指示** はやっぱり文字のほうが伝わる
- **長文** は音声入力より速い (慣れたタイピングで)
- **コードや数式** は音声では絶対打てない
- **オフィスや図書館で**音声を発しづらい

つまり、**AI時代でもキーボードはむしろ重要性が増しています**。
AI に出すプロンプトが、思考の質を決めるからです。

## まとめ

- マウスより**キーボードのほうが速い** (実証済み)
- 1980年代から「**ホームポジションから手を離さない**」文化が育った (Vim, Emacs)
- ターミナルには独自のキー文化 (Ctrl+R など)
- QWERTY 配列は **150年前の遺産**
- AI 時代でも、**キーボード = 思考を伝える最速の道具**
