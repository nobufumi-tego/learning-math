# コラム 03: なぜ世界の道具がタダで配られているのか — オープンソースの物語

> *Linux も Python も VS Code も Blender も、ぜんぶタダ。なんで?*

## 「タダ」のソフトの世界

あなたが今このリポジトリを読むのに使っているもの、ちょっと数えてみましょう:

- **Linux / Mac の OS の中核** — UNIX/Linux 系
- **ブラウザ** — Chrome (Chromium)、Firefox は両方オープンソース
- **Python** — 完全に無料
- **NumPy・JAX・matplotlib・Jupyter** — 全部タダ
- **uv** — タダ
- **Git** — タダ
- **VS Code** — タダ (Microsoft 製なのに!)

会社で言えば、**何百万円もするツール**が、全部無料。
**どうやってこの世界が成り立ってるんでしょうか?**

これを理解するには、**1980年代のあるエンジニアの怒り**から話を始める必要があります。

## リチャード・ストールマン、激おこ事件 (1980年代)

舞台はマサチューセッツ工科大学 (MIT) の AI ラボ。
**リチャード・ストールマン** (Richard Stallman、通称 RMS) という超天才ハッカーがいました。

ある日、研究室のプリンタが壊れます。修理しようとして RMS は驚く:
**プリンタのソースコードが、メーカーから提供されていない**。
「プリンタの動作を直したいから、ソースをくれ」と頼んでも「**営業秘密**」と拒否される。

これが RMS を激怒させました。
「**コンピュータの動作が、ユーザーに秘密にされている世界**」を、彼は受け入れられなかった。

## GNU プロジェクトの始まり (1983年)

1983年、RMS は壮大な計画を発表します:

> **「すべてのソフトウェアが自由でなければならない」**

そして、**全くゼロから自由な OS を作る** プロジェクトを始めました。
名前は **GNU** (グニュー)。
GNU's Not Unix の略 (再帰的略語、ハッカー流のジョーク)。

RMS は、世界中のプログラマーに呼びかけます:
**「ソフトウェアは、自由であるべきだ。みんなで、無料の道具を作ろう」**

## Free Software の "Free" の意味

「Free」というと「無料」と訳されがちですが、RMS の言う Free は **「自由 (freedom)」** の意味です。

具体的には**4つの自由**:
1. **自由 0**: 目的を問わずプログラムを実行する自由
2. **自由 1**: ソースコードを読み、動作を理解する自由
3. **自由 2**: 隣人を助けるためにコピーを再配布する自由
4. **自由 3**: プログラムを改良し、改良版を公開する自由

「**Free as in free speech, not as in free beer**」(言論の自由のフリーであって、無料ビールのフリーじゃない)
と RMS は言います。

## GPL ライセンス — 自由を強制する仕組み

RMS は単に「自由なソフトを作ろう」と言うだけでなく、**自由を制度化** しました。
それが **GPL** (GNU General Public License)。

GPLはこう言います:
> このソフトを使ってもいい。自由に改造してもいい。配布してもいい。
> **ただし、改造版を配布するときは、改造版も GPL で公開しなければならない**。

これは「**自由が伝染する**」設計。**コピーレフト** とも呼ばれます。
copy**right** (右) ではなく copy**left** (左)。
言葉遊びだけど、**思想を表す名前**です。

これによって、自由なソフトが**閉じた商品に取り込まれて消える**ことを防いでいます。

## Linux の登場 (1991年)

GNU プロジェクトはエディタ (Emacs)、コンパイラ (GCC)、シェル (Bash) などを次々と作りましたが、
**OS の中核 (カーネル) だけは、なかなか完成しませんでした**。

そんな1991年、フィンランドの大学院生 **リーナス・トーバルズ** (Linus Torvalds) が、趣味で OS のカーネルを書き始めます。
彼は学内の掲示板に、こう書き込みました:

> **"Hello everybody out there using minix - I'm doing a (free) operating system (just a hobby, won't be big and professional like gnu) for 386(486) AT clones."**
>
> (やぁみんな、minix を使ってる人。趣味で（無料の）OSを作ってるんだ。GNUほど大きくも立派にもならないけどね。386/486 AT クローン用)

これが **Linux カーネル** の誕生。
「**そんなに大きくならないと思う**」と書いていますが、結果は史上最大の OS プロジェクトに。

リーナスは Linux を **GPL** で公開します。
そしてGNU プロジェクトのツール群と組み合わさって、**完全に自由な OS** が完成しました。

正式名称は **「GNU/Linux」**。
ただ、世間は短く **「Linux」** と呼びます (RMS は今でも「GNU/Linuxと呼べ!」と訴えています)。

## オープンソース革命 (1990年代後半)

1998年、**エリック・レイモンド** という人が「Free Software」 という言葉を **「Open Source」** に言い換える運動を始めます。

理由: ビジネス世界で「Free」 は「無料」と誤解されて敬遠される。
「Open Source (開かれた源コード)」のほうが企業に受け入れられやすい。

これが大成功。
1998年に **Netscape** (ブラウザ会社) がソースを公開、それが **Mozilla → Firefox** に。
2000年代に IBM が Linux を支援、Google が Linux で検索エンジンを構築。
**「無料の道具で、世界規模のビジネス」** が当たり前になりました。

## 現代のオープンソース

今や、**世界の主要なソフトウェアの多くがオープンソース**:

| 分野 | 代表的なオープンソース |
|---|---|
| OS | Linux, FreeBSD |
| ブラウザ | Chromium, Firefox |
| サーバー | nginx, Apache |
| データベース | PostgreSQL, MySQL, SQLite |
| プログラミング言語 | Python, Ruby, Rust, Go |
| ML フレームワーク | PyTorch, TensorFlow, JAX |
| エディタ | VS Code, Vim, Emacs |
| ターミナル | bash, zsh, fish |
| Web フレームワーク | React, Vue, Django, Rails |
| インフラ | Docker, Kubernetes |

これらは**全部、無料**。
**インターネットの基盤の8〜9割は、誰かが無償で書いたコード**でできています。

## 「タダなのに、なぜ続くのか」

最大の謎: **なぜ無料で続くのか**。

理由は複数あります:

### 1. 楽しいから
最初は趣味で書いている人が多い。
「**自分が欲しいから作る**」「**他人に喜ばれるのが嬉しい**」という、純粋な動機。

### 2. 名声と評価
オープンソースで有名になると、就職・転職で**圧倒的に有利**。
GitHub のスター数が、技術者の履歴書代わり。

### 3. 大企業のスポンサーシップ
Google・Microsoft・Meta・IBM・Amazon が、社員に**業務時間でオープンソースを書かせる**ことが多い。
理由: 自社のインフラもオープンソースに依存しているから、それを保守するのは合理的。

### 4. 集合知の方が強い
**世界中の何千人**がレビュー・改善するコードと、**社内の数十人**が書くコードでは、品質が違う。
オープンソースの方が、**速く・正確に・優れた**ものになる。

### 5. 「この道具がないと困る」社会的合意
Linux やPython がなくなったら、現代社会のサービスの大半が止まります。
だから誰かが必ず維持する。

## 「タダ」 の限界 — 持続可能性問題

ところがオープンソース、いいことばかりじゃありません。

### 燃え尽きるメンテナー
「**たった1人の個人**」がメンテしている重要なライブラリが、世界には何百もあります。
その人が病気になったり、燃え尽きたり、亡くなったりすると、**世界中のサービスが影響**を受けます。

2014年の **Heartbleed バグ** (OpenSSL の脆弱性) は、世界中のサーバーを脅かしましたが、
OpenSSL のメンテナーは数人で、年間予算は **2,000ドル** (たった30万円程度) でした。

### 報酬問題
最近は GitHub Sponsors、Patreon、Open Collective などで、メンテナーに寄付する仕組みが広がっています。
それでも、**プロのフルタイムの仕事** にできる人はごく一部。

### 大企業の "ただ乗り" 批判
オープンソースを使って数兆円稼ぐ企業がある一方、メンテナーに何も還元しないことも多い。
最近 Redis や Elastic などが**ライセンス変更** で対抗する動きがあります。

## あなたができること

オープンソースの世界に少しでも還元したいなら:

- **GitHub で Star を押す** — タダでできる応援
- **バグ報告 (Issue) を書く** — メンテナーが助かる
- **ドキュメントの誤字を直す** — Pull Request の第一歩
- **使ってる道具を SNS で紹介する** — 知名度アップ
- **GitHub Sponsors で月額寄付** — 月 100円から

「**自分はプログラマーじゃないから関係ない**」と思わなくて大丈夫。
**ユーザーとしての存在自体が、その道具に価値を与えています**。

## まとめ

- 1983年、ストールマンが「**自由なソフト運動**」を始めた (GNU プロジェクト)
- 1991年、リーナスが Linux カーネルを書いた
- GPL という「**自由を強制するライセンス**」が、自由を守ってきた
- 今や、世界の主要ソフトの大半がオープンソース
- 動機は **楽しい・名声・スポンサー・集合知**
- でも**メンテナーの持続可能性**は課題

あなたが今、ペンタと一緒に学んでいるこのリポジトリも、その**オープンソース文化の上**に成り立っています。

ストールマンとリーナスに、ちょっと感謝の気持ちを抱きながら、ターミナルを開いてみてください。

```
       ／￣￣＼
      |  ・ω・  |   ぼくが Linux 出身なのも、
      |   ⌒    |   この自由な文化のおかげなんだ。
       ＼____／
```
