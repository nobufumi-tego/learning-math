# コラム 05: 今のAIは、どの数学で動いているか

> *ChatGPT が文章を生成するとき、内部で何が起きているか — 数学の側から覗いてみる。*

## AI が「賢い」のは、巨大な掛け算と足し算をしているから

驚くかもしれませんが、現代の AI (ChatGPT、Claude、Geminiなど)の本体は、**死ぬほど大量の「掛け算と足し算」をしているだけ**です。

具体的には:
- 入力 (例: 「明日の天気は」) → 数字の列に変換
- それを **重み (weight)** という巨大な行列と掛け算
- 結果に **バイアス** を足す
- それを次の層に渡して、また掛け算と足し算

これを何百層も繰り返した結果、「**晴れ**」「**雨**」などの単語が確率付きで出てきます。

「**え、それで賢く見えるの？**」と思うかもしれませんが、**何千億回もこれを繰り返す**ことで、人間と区別がつかない文章が生成されてしまう。
**それが現代AIの不気味な真実**です。

## AIで使われている数学の4本柱

### 🥇 1位: 線形代数 (Linear Algebra)

AI のあらゆる計算は **行列の積** で表現されます。

```
出力 = 入力ベクトル × 重み行列 + バイアス
```

GPT-4 の規模になると、**1750億個のパラメータ（重み）** がある巨大な行列演算を、文章1文字生成するたびに行います。

**「行列の積」は、19世紀には完成していました**。
それが21世紀になって、コンピュータの計算力に追いついて、爆発的に役立っているわけです。

### 🥈 2位: 微積分 (Calculus)

AI の **学習**(訓練)は、**勾配 (gradient)** を計算して、それに沿ってパラメータを少しずつ更新することで進みます。

「**勾配を計算する**」とは、「**この重みを少し変えたら、誤差はどう変化する？**」という質問に答えること。これがまさに**微分**です。

```
新しい重み = 古い重み − 学習率 × 勾配
```

1750億個のパラメータすべてに対して、何兆回もこの計算を繰り返す。
**Newton と Leibniz が17世紀に発明した微積分**が、現代AIの心臓部です。

### 🥉 3位: 確率・統計 (Probability & Statistics)

AI が **「明日の天気は晴れ」と答える** とき、それは:
- 「晴れ」: 70%
- 「曇り」: 20%
- 「雨」: 8%
- 「雪」: 2%
- ...

という**確率分布**から、1つを選んでいます。
これを **「次のトークン (単語)」の予測**と言います。

ChatGPT は何をしているか、本質を一言で言うと:
> **「次に来る単語の確率分布を、次々と予測している」**

それだけ。
それでも、シェイクスピアのような文章も、コードも、詩も書ける。
**確率の魔法**です。

### 4位: 三角関数 (Trigonometry)

意外な常連、**三角関数**。
特に **Positional Encoding (位置エンコーディング)** で使われます。

文章「**私はりんごが好き**」を AI に入れるとき、**「私」が1番目、「は」が2番目…** という**順序情報**を、`sin` と `cos` の組み合わせで埋め込みます:

```python
PE(pos, 2i)   = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))
```

つまり、ChatGPTが「**言葉の順番**」を理解できるのは、**三角関数のおかげ**。
こんなところで sin/cos が活躍するのは、設計者の発明的なアイデアでした。

## Transformer — 現代AIの設計図

2017年、Google の研究者たちが「**Attention is All You Need**」という論文を発表します。
ここで提案された **Transformer** という構造が、ChatGPT・Claude・Gemini を含む現代AIすべての基礎になりました。

Transformer の中心 **Attention (注意機構)** は、こういう式です:

```
Attention(Q, K, V) = softmax(Q × K^T / √d) × V
```

- `Q, K, V`: 行列
- `softmax`: 確率分布に変換する関数
- `√d`: スケーリング定数

これを見て「何が起きているか」を読み解けるようになるのが、このリポジトリの**ゴール**です。

**この式の各要素は、すべてこのリポジトリで学べる範囲のことです**:
- 行列の積 → `01_linear_algebra/`
- ソフトマックス → `06_ml_math_bridge/`
- 確率 → `03_probability_statistics/`
- 学習（微分） → `02_calculus/`

つまり、**いま読んでいるこのリポジトリの内容を全部理解できれば、最新AIの中身がだいたい分かるようになります**。

## 「シンギュラリティ」は数学者から見るとどう見えるか

世間では「**AI が人間を超える**」「**シンギュラリティ**」と騒がれます。
でも数学的に見ると、現代AIは:

- **線形代数** (19世紀完成)
- **微積分** (17世紀完成)
- **確率論** (17-19世紀完成)
- **情報理論** (1948年シャノン)
- **誤差逆伝播法** (1986年に再発見)
- **Transformer** (2017年)

**ほぼ全部、過去の数学の積み上げ**です。
革新は、計算機性能(GPU)とデータ量(インターネット)が、これらの数学が要求する規模に追いついたこと。

つまり「**AIが知っているものは、人類の数学者が用意したもの**」。
あなたが数学を学ぶというのは、**AIを作っている側の人類の知識**に近づくことです。

## AIに「使われる」のか、「使う」のか

数学を学ばないままAIを使うのは、**ブラックボックスを使う**ことです。
- AI が嘘をついても、見抜けない
- AI に何ができて何ができないかが、分からない
- AI に良い指示が出せない
- AI に騙されるリスクがある

数学を学ぶと:
- AI の限界が分かる（**確率モデル**なので、「絶対に正しい」は無い）
- AI が答えに自信があるか分かる（**ソフトマックスの分布**の形）
- AI に効率的な指示が出せる（**プロンプトエンジニアリング**は実は確率を制御している）

つまり数学を学ぶことは、**AI時代を自分の意志で生きる**ための、最も汎用的なスキルかもしれません。
